The next panel shows the player settings. It allows setting the maximum value for either the internal speakers. If these are lower than 15, the player's volume knob won't allow setting higher volume levels. Maximum brightness determines the brightness of the LED strip. The lower the level, the less battery is drained. All features but the MP3 player can be turned on/off. Note that turning the walkie-talkie mode off does not disable the WebUI, it only means that the walkie talkie player controls won't work and the player will not check for new messages. The timezone offset (in hours) can be specified for proper timekeeping. The player attempts to synchronize the time and walkie-talkie messages each day. The our at which this happens can be specified in the last input of the panel.

The player can also work as an alarm clock. The alarm can be enabled specifically for selected days and an MP3 file to be played can be selected. When the alarm sounds, short press of any button will snooze the alarm for 5 minutes, while long press will stop the alarm and either turn player off immediately, or return to the previous mode if the player was on. Stopping the alarm does not disable it so it will ring again when the conditions are met.

The player supports up to three different greetings. A greeting is specified by providing a day and month as well as an mp3 file. Any time a player is turned on that day, the selected file will be played first, then the music mode will be entered. Short press while playing the greeting message will dismiss it. This is useful for birthdays, etc.
